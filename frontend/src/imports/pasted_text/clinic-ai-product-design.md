--------------------------------------------------
REAL WORKFLOWS YOU MUST UNDERSTAND
--------------------------------------------------

1. Marketing → signup flow
A clinic owner visits the landing page, understands the product, signs up, and enters onboarding.

2. Onboarding
The clinic configures:
- clinic info
- services
- availability / hours
- FAQs
- integrations
- branding / launch readiness
Then they preview and eventually go live.

3. Patient interaction
Patients interact through:
- website chat
- chat widget
- SMS (operationally implied in product)
The assistant handles questions and booking-related intake.

4. Staff operations
Clinic staff use the dashboard to:
- view inbox
- review conversations
- manage leads
- manage appointments
- manage customers
- recover opportunities
- check operations/readiness
- monitor activity
- train the AI
- update settings
- manage billing/account

5. Launch / operations
The product includes states like:
- not live
- ready to go live
- live
- incomplete setup
- needs review
- pending billing / usage attention
- active / inactive statuses

Design must support these states clearly.

--------------------------------------------------
SPECIAL UX RULES
--------------------------------------------------

Use popups/modals only where they improve workflow.
Good places:
- delete / destructive confirm
- save confirmation
- go live
- change plan
- invite team member
- add/edit note
- add/edit training item
- test assistant
- leave with unsaved changes

Do not overuse popups.

Design all important states:
- empty
- loading
- error
- success
- incomplete setup
- blocked
- live / not live
- ready to go live
- no conversations yet
- no leads yet
- no appointments yet
- no docs uploaded yet
- billing warning / over-limit
- disabled integration
